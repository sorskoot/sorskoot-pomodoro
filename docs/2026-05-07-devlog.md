## Introduction
🚀 This post walks you through the end‑to‑end creation of the **sorskoot‑pomodoro** web app, from project bootstrap to production‑ready build. You’ll see how Vite, TypeScript, and React intertwine to deliver a fast development experience. The guide is written for developers who want a clear, minimal‑change roadmap. Each section highlights the key decisions that keep the codebase tidy and maintainable. 🎯

## Project Initialization
🛠️ We start by scaffolding a Vite + React + TypeScript template using `npm create vite@latest`. The command generates a `vite.config.ts` that sets the base path to `'./'`, ensuring correct asset resolution. TypeScript’s strict mode is enabled in `tsconfig.json` to catch early errors. This foundation gives us hot‑module reloading and an instant dev server. The result is a lightweight repo ready for feature work. 🚀

### Tooling Choices
📦 Vite handles bundling, while Vitest provides a Jest‑compatible test runner. React 18 introduces concurrent features that we leverage for responsive UI updates. ESLint with the Airbnb profile enforces consistent code style across the team. Prettier integrates automatically via VS Code to keep formatting uniform. These tools together create a robust, opinionated stack. 🔧

## Folder Overview
📁 The source tree mirrors a typical React application but is organized by feature domains. Top‑level folders like `components`, `hooks`, `pages`, and `services` group related logic. Each component folder contains a TypeScript file, a CSS module, and an index barrel for clean imports. This structure scales well as the app grows. 📂

### Key Files
🔑 `main.tsx` boots the React root and wraps it with the `ThemeProvider`. `App.tsx` defines the high‑level layout using the `Layout` component. `vite.config.ts` configures the dev server and build options. `package.json` scripts expose `dev`, `build`, and `test` commands for daily workflow. Understanding these files is essential for navigating the project. 🗂️

## Build and Test Pipeline
✅ Running `npm run build` produces an optimized static bundle in `dist/`. The build respects the Vite base setting, ensuring assets load correctly when served from a sub‑directory. `npm run test` executes Vitest, covering unit tests in the `test/` folder. Coverage reports are generated automatically to verify quality gates. This pipeline guarantees that every commit is both buildable and tested. 📊

These scripts are the primary entry points for development, production builds, and test execution. They are documented in `README.md` for newcomers. Using them ensures consistency across environments. 🛠️
